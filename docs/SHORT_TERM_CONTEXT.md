# 短期上下文管理

短期上下文只描述当前会话中下一次模型请求会携带的内容，不会跨会话保存为长期记忆。

## 上下文组成

当前请求按以下顺序组装：

1. 基础系统提示词
2. Personality
3. 长期记忆
4. 固定到本会话的消息
5. 最近 N 轮普通对话

## 消息级控制

每条完整消息都可以：

- 加入或排除上下文
- 固定到本会话

排除消息不会删除历史记录。固定消息不受最近 N 轮裁剪影响，但只在当前会话内生效。

## 清除当前上下文

“清除当前上下文”只设置一个消息边界：

- 历史消息继续保留并可浏览
- 边界之前的普通消息不再发送给模型
- 固定消息继续生效
- Personality 与长期记忆继续生效

## Token 预算

上下文检查器显示：

- 当前输入 Token 与模型上下文上限
- 最大输出 Token 预留
- 最坏情况请求预算
- 当前可用空间
- 基础提示词、Personality、长期记忆、固定消息和最近对话各自的 Token
- 每个组成部分占当前输入 Token 的百分比

顶部进度条表示“当前输入 Token ÷ 上下文上限”；最坏情况预算以辅助信息展示；组成进度条表示“该部分 Token ÷ 当前输入 Token”。

所有 Token 数均为本地估算值，实际用量以模型 API 返回结果为准。
