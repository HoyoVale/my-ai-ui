# 会话与短期上下文

## 数据文件

会话保存在 Electron 的 `userData` 目录：

```text
conversations.json
```

当前数据版本为 `4`。助手消息除正文外，还可以保存生成耗时、可展示的思考摘要与工具调用记录：

```js
{
  version: 4,
  currentConversationId: "...",
  conversations: [
    {
      id: "...",
      title: "...",
      contextStartAfterMessageId: null,
      createdAt: 0,
      updatedAt: 0,
      messages: [
        {
          id: "...",
          role: "user" | "assistant",
          content: "...",
          status: "complete" | "aborted",
          includeInContext: true,
          pinnedToContext: false,
          createdAt: 0,

          // 仅助手消息可选
          durationMs: 1200,
          reasoningSummary: "...",
          toolCalls: []
        }
      ]
    }
  ]
}
```

旧版会话会在读取时自动补齐缺失字段。

## Conversation 窗口

窗口采用轻量双栏布局：

- 顶栏只保留侧栏、上下文、新建、继续对话和窗口控制图标。
- 侧栏仅显示会话标题，不显示消息预览、更新时间和数量。
- 用户消息使用轻量气泡，助手消息采用无气泡阅读布局。
- 助手消息支持 Markdown、GFM 表格、代码块和独立复制。
- 最新助手回复支持重新生成，重新生成会替换原回复，不重复写入用户消息。
- 助手回复上方可折叠显示生成耗时、思考摘要和工具调用记录。

## 短期上下文

每次发送消息时，只取当前会话最近 N 轮完整消息。

规则：

- 当前用户消息总是包含。
- 完整助手回复进入后续上下文。
- 被停止的部分回复可以保存，但不会进入后续上下文。
- 不同会话之间不会共享消息。
- 排除的消息保留在历史中，但不进入模型请求。
- 固定消息不受最近 N 轮裁剪影响。
- 重置上下文只移动边界，不删除历史。

## Token 预算

上下文检查器显示本地估算的总 Token、输入与输出预留，并按基础提示词、Personality、长期记忆、固定消息和最近对话拆分。

组成长条表示各部分占当前输入 Token 的比例；顶部主长条表示当前输入占模型上下文上限的比例。最大输出与最坏情况请求预算作为辅助信息单独显示。

## 消息操作与会话重命名

- 用户消息上方显示发送时间，底部仅提供复制。
- 助手消息底部提供复制、重新生成、固定到本会话、加入/排除上下文；这些按钮仅在鼠标悬停或键盘聚焦时显示。
- “加入上下文”表示消息允许进入普通短期上下文，仍受最近轮数与预算裁剪影响。
- “固定到本会话”表示消息进入会话固定区，不受最近轮数与上下文重置边界影响；排除上下文时会自动取消固定。
- 会话列表悬停后可点击编辑图标重命名，也可以双击标题进入编辑；Enter 保存，Escape 取消。
